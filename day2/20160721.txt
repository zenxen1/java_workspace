1. 자바의 개발 플랫폼(자바 언어로 개발할 수 있는 분야)
javaSE : 응용 프로그램(실행할 수 있는 코드가 반드시 유저)
javaEE : 기업용 어플리케이션(웹)
javaME : 임베디드 어플리케이션 --> 안드로이드로 대체

3. 자바와 자바스크립트 차이점
               java             javascript
----------------------------------------
컴파일 여부:    o                   x
속도            o(컴파일언어)       x(인터프린터)

4. 컴파일이란? 원본소스를 기계어로 변환하는 과정

5. 자바의 슬로건인 Write once run anywhere 의 의미

 -한번 작성된 자바 프로그램은 해당 플랫폼에 독립적으로 수행될 수 있다.
 -OS 종류에 상관없이 돌아간다!

6. 바이트코드와 자바 가상머신
   바이트코드란? .class 를 의마하며 오직 JVM만이 해석가능
   java Virtual Machine : 자바 코드 해석 및 메모리를 관리 하는 소프트웨어

7.환경변수란 무엇이고 사용 목적은??
  OS에 등록하여 사용할 수 있는 변수를 의미하며 특히 path라는 변수에서 OS가 가동되고
  있는 한 언제든 접근이 가능한 전역 데이터를 등록할 수 있다.
  어느 위치에서건 등록된 경로를 참조할 수 있다. 

8. 자료형이란 무엇인고, 자바의 자료형의 종류 및 크기는?
  데이터의 종류를 의미하여 종류를 구분하는 이유는 메모리에 차지할 데이터의 용량을
  결정하기 위함이다!!

  문자 : char(2)
  숫자 : byte(1) <short(2) <int(4) <long(8)
  논리값 : boolean(1)
